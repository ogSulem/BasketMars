## 1. Описание проекта

**BasketMars** — мобильная аркадная игра для Android, реализующая механику броска баскетбольного мяча в кольцо. Управление — жест свайп по экрану, физика — параболическая траектория с обработкой попаданий по хитбоксу. Приложение написано на чистой Java без игровых движков: весь рендеринг, физика и анимации реализованы напрямую через Android Canvas API.

### Ключевые особенности

- 🏀 **3 игровых режима** — Аркада, На время (60 сек), Дуэль против ИИ-бота
- 🎨 **Инвентарь** — 3 скина мяча, 3 сетки кольца, 3 фона; разблокируются через достижения
- 🏆 **Достижения** — 8 уровней за набор очков, с визуальными наградами
- 📊 **Лидерборд** — локальная таблица (Room/SQLite), автосинхронизация с Firebase Firestore при наличии аккаунта
- 🎵 **Встроенный музыкальный плеер** — 2 фоновых трека с кнопками переключения
- 🌐 **Два языка** — Русский и English, переключение в настройках без рестарта приложения
- 🔐 **Авторизация** — Google Sign-In + Firebase Auth
- 🤖 **Человекоподобный ИИ-бот** — переменная точность, банковские броски, «холодные серии»
- 🎊 **Конфетти** — анимация при победе в дуэли

## 2. Архитектура приложения

Приложение построено по **многослойной (layered) архитектуре**:

```
┌──────────────────────────────────────────────────────────────────┐
│                        UI Layer (View)                            │
│  MainActivity · GameActivity · LeaderboardActivity                │
│  InventoryActivity · AchievementsActivity · SettingsActivity      │
├──────────────────────────────────────────────────────────────────┤
│                  Game Engine (GameView.java)                       │
│  Canvas рендеринг · Физика мяча · Жесты (swipe) · Анимации       │
│  Комбо-система · HUD · Бот-анимация · Конфетти                   │
├──────────────────────────────────────────────────────────────────┤
│               Network Layer (net/)                                │
│  MatchClient (interface) · DemoDuelClient (ИИ-бот)               │
├──────────────────────────────────────────────────────────────────┤
│                   Data Layer (data/)                              │
│  Room Database · LeaderboardRepository · CloudLeaderboardRepository│
│  LeaderboardEntry · PlayerStats · AppDatabase                    │
├──────────────────────────────────────────────────────────────────┤
│              Firebase / Google Services                           │
│  Firebase Auth (Google Sign-In) · Firebase Firestore             │
└──────────────────────────────────────────────────────────────────┘
```

**Ключевые принципы:**
- **Repository Pattern** — `LeaderboardRepository` скрывает Room, `CloudLeaderboardRepository` скрывает Firestore
- **Interface-driven networking** — `MatchClient` легко подменяется между ИИ-ботом и реальным WebSocket-сервером
- **Single-process lifecycle** — `BasketballGameApp` управляет общими ресурсами (БД, музыка)
- **Locale wrapping** — `LocaleHelper.wrap(base)` в `attachBaseContext` всех Activity обеспечивает локализацию

## 4. Компоненты и модули

### 4.1 Игровой движок (GameView)

`GameView.java` (~2600 строк) — ядро проекта. Реализует:

| Подсистема | Описание |
|---|---|
| **Физика мяча** | Параболическая траектория через Bezier-кривую; скорость, направление и пик вычисляются из вектора свайпа |
| **Детекция попадания** | Хитбокс-кольцо с проверкой вхождения мяча сверху под допустимым углом |
| **Анимации** | Дрожание кольца при промахе, сетка с физикой (spring-симуляция), анимация вхождения мяча, bounce-эффект |
| **ИИ-анимация бота** | Визуальный бросок призрака по параболе, линия свайпа, банковские броски через смещённую точку Bezier |
| **HUD** | Счёт игрока и бота, таймер (режим TIMED), комбо-полоса, статус дуэли |
| **Скины** | Процедурный рендеринг мяча (3 варианта), кольца (3 варианта), фоновый градиент |
| **Конфетти** | Частицы при победе в дуэли (ConfettiView) |

### 4.2 ИИ-бот (DemoDuelClient)

Бот имитирует поведение живого игрока:

- **Шанс промаха:** базовый 47%, минимальный 35% (не пробивает выше 65% попаданий)
- **Серии:** после 2+ попаданий подряд — «холодная серия» (4–8 обязательных промахов, вероятность 55%)
- **Типы бросков:** прямой, от левой стенки, от правой стенки (28%)
- **Позиции:** левая сторона (28%), правая (34%), центр (38%)
- **Адаптация темпа:** EWMA-сглаженная скорость броска игрока влияет на интервалы бота

### 4.3 Слой данных (Data Layer)

| Класс | Описание |
|---|---|
| `AppDatabase` | Room DB v1, 2 таблицы: `leaderboard_scores`, `player_stats` |
| `LeaderboardRepository` | CRUD для локального лидерборда (Room), запросы в bg-потоке |
| `CloudLeaderboardRepository` | Синхронизация с Firebase Firestore (только при авторизации) |
| `PlayerStats` | totalGames, arcadeBest, timedBest, duelBest, duelWins, duelLosses |

### 4.4 Авторизация (AuthManager)

Singleton. Реализует Google Sign-In → Firebase Auth. При входе:
- Имя аккаунта подставляется как имя игрока (если ещё не задано)
- Firestore-запись результата привязывается к userId

### 4.5 Локализация (LocaleHelper)

`LocaleHelper.java` — утилита для смены locale без перезапуска ОС:
- Сохраняет выбранный язык в SharedPreferences (`basketball_locale`)
- Оборачивает `Context` через `attachBaseContext` во всех 6 Activity
- При смене языка перезапускает приложение (FLAG_ACTIVITY_CLEAR_TASK)

## 5. Игровые режимы

### 🕹️ Аркада (ARCADE)
- Бесконечный набор очков
- Кольцо перемещается после каждого попадания
- Сложность возрастает с ростом счёта (кольцо уменьшается, скорость перемещения растёт)
- Комбо-система: серия попаданий даёт бонусные очки
- Результат сохраняется в лидерборд

### ⏱️ На время (TIMED)
- 60 секунд, максимум очков
- Таймер отображается на HUD
- По окончании — сводка результата + сохранение в лидерборд

### 🤖 Дуэль (ONLINE_DUEL)
- Игра 60 секунд против ИИ-бота (DemoDuelClient)
- Рядом с кольцом движется «призрак» — визуальное отображение бота
- Бот бросает мяч по-человечески: промахивается, делает банковские броски, устаёт (cold streak)
- Определяется победитель/ничья → анимация конфетти при победе
- Результат (победа/поражение/ничья) сохраняется в статистику

> **Примечание:** режим WebSocket-онлайн реализован через `OnlineMatchClient` (интерфейс `MatchClient`). Для полноценной онлайн-игры нужен WebSocket-сервер; при его отсутствии автоматически включается бот.

## 6. Игровая механика и физика

### Жест свайп
1. `ACTION_DOWN` — фиксируется начальная точка и время нажатия
2. `ACTION_UP` — вычисляется вектор броска `(dx, dy)` и скорость (`velocity = dist / dt`)
3. Из вектора вычисляются: `targetX`, `targetY` (точка кольца), `peakY` (высота дуги)
4. Мяч летит по квадратичной кривой Безье: `P(t) = (1-t)² P₀ + 2(1-t)t Pₘᵢ𝒹 + t² P₁`

### Определение попадания
- Хитбокс кольца — эллипс в плоскости `rimYCenter`
- Мяч должен пройти через хитбокс **сверху** (скорость по Y > 0 в момент пересечения)
- Допустимый горизонтальный разброс: `|ballX - hoopCenterX| < hoopWidth * 0.48`

### Усложнение в Аркаде
- Каждые 5 очков уменьшается `hoopWidth` на 4% (минимум 60% от исходного)
- Каждые 10 очков кольцо начинает перемещаться быстрее

## 7. Система достижений и инвентарь

### Достижения (8 уровней)

| Уровень | Очков | Награда |
|---|---|---|
| 1 | 5 | Мяч «Стрит» |
| 2 | 10 | Сетка «Стрит» |
| 3 | 15 | Мяч «Легенда» |
| 4 | 20 | Сетка «Легенда» |
| 5 | 25 | Все мячи |
| 6 | 30 | Все сетки |
| 7 | 35 | Синий фон |
| 8 | 45 | Оранжевый фон |

Уровень достижений хранится в `SharedPreferences` (`achievementLevel`). Проверка и выдача — в `GameView` после каждого попадания.

### Инвентарь

Выбранные скины сохраняются в SharedPreferences:

| Ключ | Значения |
|---|---|
| `selectedBall` | 0 = Обычный, 1 = Стрит, 2 = Легенда |
| `selectedHoop` | 0 = Обычная, 1 = Стрит, 2 = Легенда |
| `selectedBg` | 0 = Розовый, 1 = Синий, 2 = Оранжевый |

## 10. Сборка и запуск

### Требования

- **Android Studio** Hedgehog (2023.1.1) или новее
- **JDK 17** (поставляется с Android Studio)
- **Android SDK** API 34
- Устройство или эмулятор с Android 5.0+

### Шаги

```bash
# 1. Клонировать репозиторий
git clone https://github.com/ogSulem/BasketMars.git
cd BasketMars

# 2. Открыть в Android Studio: File → Open → выбрать папку BasketMars

# 3. Синхронизировать Gradle (автоматически или File → Sync Project)

# 4. Запустить: Run → Run 'app'  (Shift+F10)
```

### Сборка release APK

```bash
./gradlew assembleRelease
# APK: app/build/outputs/apk/release/app-release-unsigned.apk
```

> **Для подписи** нужен keystore. Настройте `signingConfigs` в `app/build.gradle` или подпишите вручную через `apksigner`.

### Firebase

Текущий `google-services.json` содержит **placeholder**-данные. Для активации авторизации и Firestore:
1. Создайте проект на [console.firebase.google.com](https://console.firebase.google.com/)
2. Добавьте Android-приложение с package: `com.example.basketballgame`
3. Скачайте `google-services.json` и замените файл в папке `app/`

### Unit-тесты

```bash
./gradlew test
# Отчёт: app/build/reports/tests/testDebugUnitTest/index.html
```

Тесты: `GameModeTest.java`, `PlayerStatsTest.java` (JUnit 4)
